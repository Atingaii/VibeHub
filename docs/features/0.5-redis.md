# 0.5 Redis 连接

## 目标

引入 go-redis v9 客户端，初始化 Redis 连接池，启动时 Ping 验证，断连自动重试，集成到 `/health` 端点。

## 不做

- 不创建 `internal/cache/keys.go`（后续业务缓存阶段做）
- 不实现 Lua 脚本 / 分布式锁
- 不实现 DB 切换（初期统一用 DB 0，后续按 ADR-003 分 DB）

## 数据模型

无新增数据库表。

## 配置读取

从 `configs/dev.yaml` → `config.RedisConfig`：

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `addr` | string | `localhost:6379` | Redis 地址 |
| `password` | string | `""` | 密码 |
| `db` | int | `0` | DB 编号 |
| `pool_size` | int | `100` | 连接池大小 |
| `min_idle_conns` | int | `10` | 最小空闲连接 |

环境变量覆盖（已在 0.2 定义）：`REDIS_ADDR` / `REDIS_PASSWORD`

## 接口设计

### 新增文件

- `internal/cache/redis.go` — Redis 客户端管理器（对齐 ADR-003 代码锚点 `internal/cache/redis.go`）

### 改造文件

| 文件 | 改动 |
|------|------|
| `main.go` | 数据库初始化后增加 Redis 初始化 + defer Close |
| `internal/server/server.go` | Server struct 增加 `redis` 字段，NewServer 接收 redis 参数 |
| `internal/server/router.go` | `/health` 增加 `redis_ok` 字段 |

## 实现步骤

1. `go get github.com/redis/go-redis/v9`
2. 创建 `internal/cache/redis.go`（NewRedis / Ping / Close / Client getter）
3. 改造 `main.go` 添加 Redis 初始化
4. 改造 `server.go` 接收 Redis
5. 改造 `router.go` `/health` 增加 redis_ok
6. 编译 + 运行时验证

## 退出门

- [x] `go build .` 编译通过
- [x] 无 Redis 时启动失败并输出清晰错误日志
- [x] Redis 可用时启动成功，日志输出 `[cache] Redis connected`
- [x] `GET /health` 返回 `redis_ok: true/false`
- [x] 连接池参数从配置文件读取（pool_size / min_idle_conns）
