# 0.4 数据库连接

## 目标

引入 GORM ORM，初始化 MySQL + PostgreSQL 双连接池。启动时验证连接，断连自动重试。`/health` 端点增加数据库连接状态检查。

## 设计

### 架构决策引用

遵循 [ADR-002](../adr/002-dual-database.md)：MySQL 存交易型数据，PostgreSQL 存内容型数据。

### 新增包：`internal/database`

提供统一的数据库初始化入口，封装 GORM 连接池配置。

#### `internal/database/database.go`

```go
// Manager 管理 MySQL + PostgreSQL 双连接
type Manager struct {
    MySQL    *gorm.DB
    Postgres *gorm.DB
}

// New(cfg *config.DatabaseConfig, logger *zap.Logger) (*Manager, error)
//   - 初始化 MySQL GORM 连接（gorm.io/driver/mysql）
//   - 初始化 PostgreSQL GORM 连接（gorm.io/driver/postgres）
//   - 配置连接池参数（MaxOpenConns / MaxIdleConns / ConnMaxLifetime）
//   - 启动时 Ping 验证连接
//   - 返回 Manager 供其他模块使用

// Close() 关闭底层连接
```

### GORM Logger 集成

使用 GORM 内置 logger 适配 zap，开发环境输出 SQL，生产环境仅输出 Warn+Error。

### Server 改造

- `internal/server/server.go:Server` 结构体增加 `db *database.Manager` 字段
- `NewServer` 接收 `*database.Manager` 参数
- `main.go` 在 logger 初始化后、server 创建前初始化数据库

### /health 端点改造

- `internal/server/router.go` 的 `/health` handler 增加数据库 Ping 检查
- 返回 `mysql_ok` + `postgres_ok` 字段
- 任一数据库 Ping 失败时 status 变为 `degraded`（HTTP 仍返回 200，便于健康检查不误杀）

### 连接重试策略

GORM 底层 database/sql 自带连接池管理和自动重连。额外配置：
- `ConnMaxLifetime` = 1h（从配置读取）
- `MaxIdleConns` / `MaxOpenConns` 从配置读取
- 启动时如果 Ping 失败，最多重试 3 次（间隔 2s），全部失败则 Fatal 退出

### 依赖

- `gorm.io/gorm`
- `gorm.io/driver/mysql`
- `gorm.io/driver/postgres`
