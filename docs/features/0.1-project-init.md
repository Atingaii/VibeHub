# 0.1 项目初始化

## 目标

创建 VibeShop 后端项目骨架：目录结构 + Gin HTTP server 空壳 + `/health` 健康检查端点。

## 边界

- **做**：目录结构、`main.go`、Gin router、`/health` 端点、`go.sum` 依赖锁定
- **不做**：配置加载（0.2）、日志框架（0.3）、数据库连接（0.4）、Redis（0.5）、Docker 基础设施（0.6）

## 目录结构

```
main.go                         # 唯一入口
internal/
  server/
    server.go                   # HTTP server 初始化 + 路由注册
    router.go                   # 路由定义（按模块分组）
  middleware/                   # 中间件（本期为空目录）
  module/                       # 业务模块（本期为空目录）
  model/                        # 数据模型（本期为空目录）
  store/                        # 数据访问层（本期为空目录）
  cache/                        # Redis 缓存层（本期为空目录）
  config/                       # 配置加载（本期为空目录）
  mq/                           # 消息队列（本期为空目录）
```

## 接口设计

### GET /health

- **请求**：无参数
- **响应**：HTTP 200
  ```json
  {
    "status": "ok",
    "service": "vibeshop",
    "version": "dev"
  }
  ```
- **用途**：Docker healthcheck、K8s readiness probe、开发验证

## 实现步骤

1. `go get github.com/gin-gonic/gin` 添加 Gin 依赖
2. 创建 `internal/server/server.go` — HTTP server 初始化
3. 创建 `internal/server/router.go` — 路由注册（含 `/health`）
4. 创建 `main.go` — 入口，调用 server 启动
5. 创建空目录占位（用 `.gitkeep`）
6. 验证 `go run .` → `curl localhost:8080/health` 返回 200
