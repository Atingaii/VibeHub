# 0.9 中间件主机端口参数化 + 容器自启

> 后写补档。本特性在 commit `d737638` (主机端口参数化) 与 `9d7cbd5` (restart 策略 + WSL 自启) 中实现，
> 当时未先写 `docs/features/`，按 AGENTS.md R1 要求事后补一份功能级设计文档。

## 目标

让 `make infra-up` 在两类常见环境压力下仍可一次成功：
1. **端口冲突**：本机 / Windows 侧已占用 3306/5432/6379/4222 的标准端口（典型场景：Win 装了 MySQL80、Ubuntu 自带 redis-server）
2. **重启延续**：WSL2 / 宿主重启后，中间件容器无需手工 `make infra-up` 重新拉起

同时把 PG 密码格式约束（URL-safe）落到文档，避免 pgx 解析 DSN 失败。

## 不做

- 不动应用进程的生命周期（开发模式下应用仍由 `go run .` / `make dev` 起）
- 不引入 systemd unit 来托管应用（生产部署走 `make docker-up` 全栈 compose）
- 不做端口冲突的"自动探测 + 自动错位"（保留显式由 `.env` 控制，避免黑盒）
- 不动容器内端口（仍为标准 3306/5432/6379/4222），只参数化主机映射端口

## 设计要点

### 端口参数化

| 变量 | 默认 | 容器内端口 | 说明 |
|------|------|-----------|------|
| `VIBESHOP_MYSQL_HOST_PORT` | `3306` | `3306` | 与 Win MySQL80 冲突时改 `3307` |
| `VIBESHOP_PG_HOST_PORT` | `5432` | `5432` | |
| `VIBESHOP_REDIS_HOST_PORT` | `6379` | `6379` | 与 host redis-server 冲突时改 `6380` |
| `VIBESHOP_NATS_HOST_PORT` | `4222` | `4222` | |
| `VIBESHOP_NATS_MON_HOST_PORT` | `8222` | `8222` | |

`docker-compose.infra.yml` 与 `docker-compose.yml` 的 ports 段统一改为 `${VIBESHOP_*_HOST_PORT:-默认}` 形式。

> **关键不变量**：容器**内部**端口保持标准。所以 `docker-compose.yml` 里 app 服务的 DSN（`mysql:3306`/`postgres:5432`/`redis:6379`/`nats:4222`）一行都不用改。
> 端口错位**只影响 host 上的客户端**（包括开发模式下本机跑的 Go 应用 + `configs/dev.yaml` 的 DSN/`addr`）。

### 容器自启

- 4 个中间件加 `restart: unless-stopped`，与同 compose 文件中 app 服务保持一致
- WSL2 环境下：
  - `/etc/wsl.conf` 必须有 `[boot] systemd=true`
  - `systemctl enable docker` 让 dockerd 跟随 systemd 启动
  - 链路：WSL 启动 → systemd PID1 → docker.service → unless-stopped 容器自动拉起

### PG 密码格式约束

pgx 驱动按 RFC 3986 URL 解析 DSN，密码段中的 `/` `+` `@` `:` 等都会破坏解析。
约定：PG 密码用 `openssl rand -hex 24`（48 hex 字符，纯 `[0-9a-f]`），其他密码可继续用 `rand -base64`。

## 验证步骤

1. **端口参数化**：`docker compose -f deploy/docker/docker-compose.infra.yml config` 看到端口被 `.env` 解释为预期值
2. **健康**：`make infra-up` 后 `docker ps --filter name=vibeshop` 全部 `(healthy)`
3. **应用连通**：`go run .` 后 `curl localhost:8080/health` 返回 `mysql_ok/postgres_ok/redis_ok=true`
4. **自启验证**：
   - `docker inspect -f '{{.HostConfig.RestartPolicy.Name}}' vibeshop-mysql` 返回 `unless-stopped`（4 个容器都验）
   - `systemctl is-enabled docker` 返回 `enabled`
   - `wsl --shutdown` 后再 `wsl`，进 WSL 直接 `docker ps` 应已看到 4 个 healthy 容器，无需 `make infra-up`
5. **doc-lint**：`make doc-lint` 0 错 0 警告

## 权衡 / 已知约束

- **端口错位是显式动作**：默认行为不变（仍用 3306 等），需要错开时由用户在 `.env` 取消注释。优点是行为可预测；代价是用户得自己判断有没有冲突
- **PG 密码 URL-safe 约束**：靠文档传达，没在代码层面强制校验。如果以后吃过这个亏，可以在 `internal/database/database.go` 连库前加一次 `url.Parse` 自检
- **应用进程不自启**：开发期由开发者手起；生产期通过 `make docker-up` 让 app 容器同样 `unless-stopped`。如果未来想让本地开发也自启应用，可以考虑加一个 systemd user unit 或 air 启动脚本，但目前不在范围内
